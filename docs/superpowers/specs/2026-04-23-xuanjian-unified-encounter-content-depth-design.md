# 玄鉴统一奇遇内容厚度设计

- 日期：2026-04-23
- 状态：已完成脑暴，待用户 review
- 作用域：`src/config/xuanjianCombat.ts`、`src/services/CultivationRewardEngine.ts`、`src/services/CombatService.ts`、`src/types/cultivationCombat.ts`、`src/types/services.ts`、相关 handler、测试与手册
- 目标：把当前仍偏 `starter` 的玄鉴奇遇战内容层，收口为“统一敌人原型库 + 统一奇遇模板库 + 混池宝物 + 可放弃争抢”的前中期完整内容框架，同时保持现有 `combat runtime` 主链路不被推翻

## 一、结论

这轮不再按 `胎息 / 练气 / 筑基` 各自维护三套敌人表和三套奇遇池，而是统一为一套可跨境界复用的内容框架：

1. `敌人` 改为 `通用原型库 + 境界实例化`
2. `奇遇` 改为 `通用模板库 + 宝物价值驱动风险`
3. `宝物` 改为 `全局混池`，低境界也可见高价值内容
4. `风险` 不再靠境界锁池，而靠守宝敌人强度和玩家选择
5. 玩家在守宝奇遇里始终拥有 `离开` 选项，离开时无伤无损，但宝物直接消失
6. 高阶法门 / 神通可以掉落入包，但在低境界只作为 `待解锁 / 待参悟条目` 保存，不会立刻变成可用战斗技能

这轮的设计重点是“内容厚度”而不是“求解器重写”。现有 `CombatStateAdapter -> CombatResolver -> CombatRewardBridge` 链路继续保留，调整主要发生在内容组织、奇遇阶段拆分和用户可见选择流上。

## 二、范围与边界

本设计覆盖：

1. 前中期奇遇战内容厚度收口
2. 敌人原型库与跨境界实例化规则
3. 奇遇模板库与守宝风险模型
4. 宝物混池、争抢与放弃语义
5. 高阶内容掉落后的“入包但不立即可用”规则
6. 对现有 encounter runtime contract 的最小扩展方向

本设计不覆盖：

1. `紫府+` 内容厚度扩展
2. 长期数值平衡和高阶构筑长期平衡
3. 守关战、破境战、PVP、长回合 `/fight`
4. 宝物线索回溯、反复追踪、失之交臂后二次再遇
5. `CombatResolver` 的大规模机制重写

## 三、Authority 与现状判断

### 3.1 这轮以当前 runtime 为准

历史文档中存在一个口径冲突：

1. 图鉴文档更倾向 `战斗法门从练气起，神通从紫府起`
2. 当前 runtime 已经允许 `胎息` 装备 starter 法门，并且在 `练气 / 筑基` 提前启用了部分 `power.*`

本轮设计明确采用：

`当前 runtime 口径优先，旧文档后续回写同步。`

也就是说：

1. 不回退现有 `胎息` starter 法门
2. 不为了追求旧文档一致性而回滚当前已接线内容
3. 后续文档同步以“当前运行时可用语义”为准

### 3.2 当前真正的问题不是“没战斗”，而是“内容层仍像 starter”

当前系统已经具备：

1. 最小 combat runtime
2. 敌方模板与奇遇战模板
3. 配装投影与战斗摘要输出

但仍存在三个内容层缺口：

1. 敌人与奇遇仍显得像少量样例，而不是完整内容库
2. 高价值宝物与高风险战斗之间缺少正式风险选择流
3. 高阶内容掉落、入包、解锁之间缺少正式语义，容易把 progression 打散

## 四、统一敌人原型库

### 4.1 设计原则

敌人不再按境界手写三套重复配置，而是维护少量高复用原型，每个原型都可实例化为 `胎息 / 练气 / 筑基` 版本。

统一实例化公式：

`enemyInstance = enemyPrototype + realmId + realmSubStageId + loadoutSeed + namingSkin`

其中：

1. `enemyPrototype` 决定战斗风格
2. `realmId / realmSubStageId` 决定强度包络
3. `loadoutSeed` 决定可挂载法门和神通
4. `namingSkin` 决定名字、外观文案和遭遇氛围

### 4.2 原型数量

本轮统一收口为 `6` 个原型：

1. `冲阵型`
2. `坚守型`
3. `游走型`
4. `灵识型`
5. `消耗型`
6. `精英混成型`

### 4.3 原型定义

#### `冲阵型`

- 目标：抢先压血线，缩短战斗
- 主要 tags：`rush` `attack`
- 优先挂载：`attack + movement`
- 皮肤示例：青狼、凶犬、悍修

#### `坚守型`

- 目标：高防承伤后反打
- 主要 tags：`guard` `defense`
- 优先挂载：`guard + support`
- 皮肤示例：獠猪、石傀、甲兽

#### `游走型`

- 目标：高速度拉扯、切角、规避正面对拼
- 主要 tags：`movement` `flank`
- 优先挂载：`movement + attack`
- 皮肤示例：山貂、狐妖、飞盗

#### `灵识型`

- 目标：识破、先手、扰乱、压制
- 主要 tags：`sense` `control`
- 优先挂载：`support + control`
- 皮肤示例：妖鸦、瞳灵、镜煞

#### `消耗型`

- 目标：拖回合、磨状态、拉低稳定度
- 主要 tags：`support` `ward`
- 优先挂载：`support + guard`
- 皮肤示例：藤魅、雾鬼、药傀

#### `精英混成型`

- 目标：同时具备两类原型能力，用于守高价值宝物
- 主要 tags：`elite` `hybrid`
- 优先挂载：两种偏好混挂
- 皮肤示例：守陵傀、异种妖、客卿修士

### 4.4 跨境界复用规则

同一个原型在不同境界不换机制，只换以下内容：

1. 四维基线
2. 小阶强度包络
3. 可挂载法门 / 神通门槛
4. 名称皮肤与描述文案

因此：

1. `胎息青狼`
2. `练气悍修`
3. `筑基裂甲妖`

可以是同一 `冲阵型` 原型的三个实例，而不是三张互不相关的敌表。

## 五、统一奇遇模板库与宝物风险模型

### 5.1 模板库原则

奇遇同样不按境界拆池，而是只维护一套通用模板库，再根据宝物价值、环境气氛和守宝敌人原型进行实例化。

本轮固定 `7` 类模板：

1. `路边可取型`
2. `守宝对峙型`
3. `兽巢藏物型`
4. `遗府残坛型`
5. `险地明宝型`
6. `伏击夺宝型`
7. `重宝镇守型`

### 5.2 模板职责

#### `路边可取型`

- 低风险
- 可无守宝者
- 主要承载“偶得小利”

#### `守宝对峙型`

- 明确展示宝物与守宝者
- 玩家二选一：`离开 / 争抢`
- 是本轮的标准选择型模板

#### `兽巢藏物型`

- 更容易挂 `兽类 / rush / guard` 原型
- 适合灵材、丹药、低中阶法器

#### `遗府残坛型`

- 更容易挂 `sense / control / elite` 原型
- 适合玉简、功法、传承类宝物

#### `险地明宝型`

- 宝物价值高，危险也直观
- 承担“高收益高风险但可放弃”的叙事职责

#### `伏击夺宝型`

- 先展示不祥征兆，再给出争抢选择
- 更偏剧情化短奇遇

#### `重宝镇守型`

- 高价值宝物默认高权重进入该模板
- 默认更容易绑定 `精英混成型`

### 5.3 宝物风险模型

本轮的正式语义是：

`宝物池混池 != 风险混同`

也就是说：

1. 所有境界共享宝物可见性
2. 高价值宝物通常对应更高风险模板和更强守宝敌人
3. 风险控制依赖玩家选择，不依赖“看不见高价值内容”

### 5.4 放弃争抢语义

当奇遇带有守宝者时，玩家始终拥有 `离开` 选项。

`离开` 的正式结果固定为：

1. 本次不进入战斗
2. 本次不受伤
3. 本次不额外掉资源
4. 本次宝物直接消失
5. 本次奇遇立即结束

本轮明确不支持：

1. 追踪
2. 线索保留
3. 下次再遇同一宝物

## 六、内容池组织与掉落语义

### 6.1 前中期核心池组织方式

本轮不按“每个境界一张独立清单”组织内容，而按：

`功能位 + 境界门槛 + 风格标签`

组织前中期内容池。

固定功能位：

1. 法门：`attack / guard / movement / support`
2. 神通：`burst / control / ward / domain`

固定风格标签：

1. `rush`
2. `defense`
3. `movement`
4. `sense`
5. `hybrid`

每个条目至少要声明：

1. 它属于哪个功能位
2. 它从哪个大境 / 小阶开始可挂载
3. 它更适合哪类敌人和哪类玩家构筑

### 6.2 前中期厚度目标

本轮的内容厚度目标不是无限扩池，而是把前中期核心池补到“够形成完整段落”。

建议目标：

1. `胎息` 核心法门 `6` 门
2. `练气` 新增核心法门 `6` 门，新增低中阶神通 `2` 门
3. `筑基` 新增核心法门 `6` 门，新增低中阶神通 `2` 门

因此，这轮收口后，前中期至少应有：

1. 核心法门 `18` 门
2. 核心神通 `4` 门

这里的“核心”指：

1. 已过 balance review
2. 已进入 runtime-ready
3. 已具备明确功能位与风格标签

不把 `pending-balance / ambiguous / quasi-divine` 计入完成度。

### 6.3 敌人与玩家共享内容池

敌人不维护独立技能宇宙，而是优先从同一内容池里抽取条目。

推荐规则：

1. `rush` 原型优先挂 `attack + movement`
2. `guard` 原型优先挂 `guard + support`
3. `movement` 原型优先挂 `movement + attack`
4. `sense` 原型优先挂 `support + control`
5. `hybrid elite` 按两种偏好混挂

这样做的目的不是省事，而是保证：

1. 世界内容统一
2. 玩家与敌人共享同一套内容语义
3. 新增一个条目时，能够同时扩充玩家构筑和敌人表现

### 6.4 高阶内容掉落后的语义

本轮明确采用：

`掉落混池 != 立即可用混池`

因此，当低境界玩家获得高阶法门 / 神通时：

1. 条目可进入背包 / 传承记录
2. 条目可显示为 `待参悟 / 待解锁`
3. 条目不可立刻进入 battle loadout
4. 条目需满足正式境界门槛后才可转为 runtime-ready 可装备内容

这个规则用于防止混池掉落直接冲烂 progression。

## 七、运行时主链与契约调整

### 7.1 主链从“结果型奇遇”改成“两段式奇遇”

当前运行时更接近：

`roll 到 encounter result -> 直接结算`

本轮应调整为：

`roll 到宝物`
-> `选择奇遇模板`
-> `判断是否存在守宝者`
-> `若无守宝者，直接获得`
-> `若有守宝者，展示风险与选择`
-> 玩家选 `离开 / 争抢`
-> `离开`：结束
-> `争抢`：进入现有 combat runtime

因此，本轮不要求重写 `CombatResolver`，但要求在 encounter runtime 中正式引入：

1. `发现阶段`
2. `选择阶段`
3. `结算阶段`

### 7.2 建议新增的运行时语义

当前 `CultivationEncounterResult` 更偏最终结果对象。为支持新流程，后续实现应允许存在“待玩家选择”的中间态。

推荐最小语义扩展：

1. `encounterOffer`
   表示当前玩家看到的宝物与风险
2. `encounterOfferStatus`
   至少区分 `available / claimed / abandoned / expired`
3. `riskTier`
   用户可见风险等级：`寻常 / 棘手 / 凶险 / 极险`
4. `guardianStyle`
   用户可见风格提示：`迅攻 / 坚守 / 游走 / 灵识 / 混成`

这层扩展的目的不是引入复杂状态机，而是让“离开 / 争抢”有正式契约。

### 7.3 用户可见层要求

当奇遇存在守宝者时，用户至少应看到：

1. 宝物名
2. 宝物品阶
3. 风险提示：`寻常 / 棘手 / 凶险 / 极险`
4. 守宝者风格：`迅攻 / 坚守 / 游走 / 灵识 / 混成`
5. 选择按钮：`离开`、`争抢`

不需要展示：

1. 精确数值
2. 精确敌方四维
3. 精确胜率百分比

本轮的目标是“信息足够做选择”，而不是“把战斗计算器直接暴露给用户”。

## 八、错误处理与异常语义

后续实现需明确以下保护规则：

1. 若守宝奇遇缺失原型或模板，回退为无守宝者的普通获得，不能让用户卡死
2. 若中间态奇遇已过期或已被处理，按钮回调必须返回用户友好提示
3. 若玩家当前状态已不满足争抢前提，必须拒绝进入战斗，并保留无伤离开兜底
4. 若高阶内容掉落后无法立即转为可用条目，必须明确展示为 `待参悟 / 待解锁`，而不是静默吞掉
5. 若 combat runtime 在争抢阶段失败，不影响“离开无伤”这一规则；两者语义必须分离

## 九、测试与验收

### 9.1 测试重点

本轮测试重点应覆盖：

1. 同一敌人原型可在不同境界实例化
2. 混池宝物可被低境界玩家遇见
3. 高价值宝物更高概率绑定高风险模板
4. 用户选择 `离开` 时无伤无损且宝物消失
5. 用户选择 `争抢` 时才进入 combat runtime
6. 低境界获得高阶法门 / 神通后只入包，不可立即配装
7. 用户可见风险和风格提示稳定输出

### 9.2 完成标准

本轮完成只看以下 `6` 条：

1. 存在一套通用敌人原型库，不再按境界手写三套重复敌表
2. 存在一套通用奇遇模板库，不再按境界拆奇遇池
3. 宝物池全局混池，低境界也可见高价值宝物
4. 高价值宝物通常绑定更危险的守宝敌人，但玩家可无伤离开
5. 高阶法门 / 神通可掉落入包，但不会在低境界立刻变成可用技能
6. 手册口径从 `starter 奇遇战` 更新为“前中期完整内容版奇遇战框架”

### 9.3 明确不承诺

这轮不承诺：

1. 前中期长期数值已经完全平衡
2. 敌我风格差异已经在求解器里完全兑现
3. `紫府+` 内容厚度已经同步完成
4. 失去宝物后的线索回溯和重复追踪

## 十、推荐的后续顺序

本设计通过后，推荐的实施顺序固定为：

1. 先重构 encounter runtime 为“两段式奇遇”
2. 再落地统一敌人原型库与模板库
3. 再补前中期核心法门 / 神通池
4. 最后更新手册、冒烟清单与回归测试

原因很简单：

如果先补大量内容，再补“离开 / 争抢”选择流，已有内容还要重接一次；反过来先把流转结构定住，后面的内容扩容更稳定。
