# 修仙系统

## 概述

修仙系统是项目的游戏化机制，通过将任务完成与修仙境界相结合，增加用户的参与度和动力。系统基于中国修仙文化，将专注任务转化为灵力修炼，实现个人成长的可视化。

## 系统架构


### 灵力计算

#### 基础灵力奖励
- 每分钟任务 = 1点灵力
- 基础倍率：根据当前境界和阶段调整

#### 境界倍率
```typescript
function calculateCultivationBonus(realm: Realm, stage: string): number {
  const baseMultiplier = 1.0;
  const stageMultiplier = {
    '初期': 1.0,
    '中期': 1.2,
    '后期': 1.5,
    '巅峰': 2.0
  };

  return baseMultiplier * (stageMultiplier[stage] || 1.0);
}
```

#### 仙石奖励
- 每分钟任务 = 0.5仙石
- 用于占卜和突破

### 核心方法

#### `awardCultivation(userId, duration)`
奖励修炼

**参数：**
- `userId`: 用户ID
- `duration`: 任务时长（分钟）

**流程：**
1. 获取用户当前修仙状态
2. 计算基础灵力和仙石奖励
3. 检查并触发仙缘事件
4. 应用奖励并更新用户状态
5. 检查境界升级

**返回：** `{ spiritualPower, immortalStones, bonus, fortuneEvent, ... }`

#### `getCultivationStatus(userId)`
获取修仙状态

**参数：**
- `userId`: 用户ID

**返回：** `{ user, ...display, immortalStones, ascensions, ... }`

### 核心方法

#### `castDivination(userId, betAmount)`
进行占卜

**参数：**
- `userId`: 用户ID
- `betAmount`: 投注仙石数量

**流程：**
1. 验证用户仙石数量
2. 随机生成卦象
3. 计算奖励
4. 更新用户灵力和仙石
5. 记录占卜历史
6. 检查境界变化

**返回：** `{ roll, gua, betAmount, result, powerChange, ... }`

#### `getDivinationHistory(userId, limit)`
获取占卜历史

**参数：**
- `userId`: 用户ID
- `limit`: 记录数量限制

**返回：** 占卜历史记录列表

#### `getDivinationStats(userId)`
获取占卜统计

**参数：**
- `userId`: 用户ID

**返回：** 占卜统计数据

## 3. 突破系统

### 突破条件

- 灵力达到当前境界最大值
- 不是大乘期（需要飞升）

### 核心方法

#### `attemptBreakthrough(userId)`
尝试突破

**参数：**
- `userId`: 用户ID

**流程：**
1. 检查突破条件
2. 计算成功率
3. 随机决定突破结果
4. 成功时：升级境界
5. 失败时：扣除灵力并可能跌落境界

**返回：** `{ success, message, oldRealm, newRealm, ... }`

## 4. 飞升系统

### 飞升条件

- 大乘期修士
- 灵力达到50000

### 核心方法

#### `ascend(userId)`
飞升

**参数：**
- `userId`: 用户ID

**流程：**
1. 检查飞升条件
2. 执行飞升
3. 增加飞升次数
4. 增加仙位印记
5. 重置境界

**返回：** `{ success, ascensionCount, immortalMarks, message }`
